How to Use the FY2026 RUS DLT Scoring + Partner Radius Tool

Use this page as the operating guide for building a seed site list, finding 50–100 mile radius partners, assigning Hub / End-User designations, and preparing a cleaner DLT scoring worksheet for grant review.

What this tool is for

Important: This is a pre-application scoring and prospecting tool. It helps organize site addresses, Census place population, County SAIPE poverty, and provisional scores. USDA’s DLT map review and final application documentation still control the official score.

Step-by-step workflow

1
Choose the project type.

Select Fixed, Hybrid, or Non-fixed. Most school district projects are Fixed unless the project also serves home-based, mobile, or changing end-user locations.

2
Select the state and default site designation.

For K-12 schools, End-User is usually the default. District offices are usually Hub unless they also receive end-user benefit.

3
Search and add the seed organization’s locations.

Start with one school district, public charter school, health center, college, or technical school. For K-12, use “Add district office + schools.” For charter schools, use the Prospect Finder Charter Schools type or K-12 school-site list and verify the legal applicant/consortium role. For healthcare, Georgia is the default state and the tool tries the HRSA XLSX source first. For higher ed, search the institution or add curated/manual locations.

4
Build a 50–100 mile partner radius list.

Choose an anchor site from the seed organization, pick 50, 75, or 100 miles, and search official K-12, HRSA healthcare, and higher-ed sources for nearby partners. Partner descriptions are assembled only from returned source fields.

5
Add only real candidate partners to the worksheet.

Use the partner list as a prospecting screen. Added partners are placed in the worksheet as not included by default so they do not inflate scoring until reviewed and intentionally included.

6
Add missing or special sites manually.

Use Add Manual Location for district offices, consortium hubs, partner clinics, extension campuses, libraries, or other sites not returned by the data source.

7
Enrich with Census + SAIPE.

Click Enrich Census + SAIPE to pull 2020 Census place population and 2024 county SAIPE poverty data where available.

8
Verify Rurality with the USDA DLT map.

Open the DLT map and update the Contiguous / Non-Rural, Outside Non-Rural, and Census Rural fields. This is the most important manual quality-control step.

9
Review the score snapshot.

Confirm the D-1 Rurality and D-2 Economic Need averages. Add a manual D-3 Need/Benefits estimate only after reviewing the narrative evidence and community need.

10
Export or print.

Export CSV for worksheet review, JSON for saving the project data, partner CSV for outreach/prospecting, or Save/Print PDF for a quick internal review package.

Scoring Tool import + auto-score workflow

Use the Scoring Tool for project-specific locations. Search or import the locations that may actually be included in one DLT application. For broad statewide prospecting, use the Prospect Finder first, then send selected records into the Scoring Tool.
  1. Keep Auto-enrich + score checked when adding or importing rows.
  2. Upload a manual site file, NCES/CCD school export, NCES/CCD district export, higher-ed/IPEDS file, or HRSA file as CSV/XLSX/XLS.
  3. The tool maps available source fields into the worksheet, then looks up Census 2020 place population and 2024 county SAIPE where possible.
  4. Review the front scoring columns: D-1 Rurality, D-2 Economic Need, 2020 Pop, SAIPE %, and D-1+D-2 Total.
  5. Before relying on the score, verify the DLT map fields and correct the Rurality Place/CDP if a mailing city is not the actual rurality community.

Healthcare workflow — GA default + HRSA XLSX fallback

Default healthcare source: The Healthcare button uses HRSA Health Center Service Delivery and Look-Alike Sites and now tries the official XLSX file first. The scoring workspace opens with Georgia selected, so leaving the search field blank will load the Georgia healthcare list first. Use a search term when you want to narrow the list by health center, clinic/site name, city, county, or address.
When live HRSA loading is blocked: Click Download HRSA XLSX, save the file locally, then open Import / Templates and upload it under Healthcare Lookup Dataset Fallback. That lookup file is separate from the Project Site List import; it only feeds the healthcare search results so you can choose which sites to add to the scoring worksheet.
When to use End-User

Use for clinics, health centers, or hospital sites where telemedicine equipment will be installed or where patients/providers will directly receive telemedicine benefit.

When to use Hub

Use for an administrative or provider-originating site that sends clinical services/content to end-user locations but does not itself receive end-user benefit.

When to use Hub/End-User

Use for a healthcare location that both originates telemedicine services and also receives equipment/benefit as a patient or provider service site.

Manual cleanup

After loading healthcare records, verify the site designation, address, county/FIPS, and whether the site is truly part of the proposed DLT project before exporting the worksheet.

Project type and site designation definitions

Fixed Project

End-user locations are specific, known physical sites where grant-funded equipment will be installed or placed. Example: a school district installing distance learning equipment in school buildings.

Hybrid Project

The project includes both fixed sites and non-fixed users. Example: clinics receive equipment and the health system also supports home-based patient telemedicine kits.

Non-fixed Project

The end-user locations change constantly, such as homes or mobile facilities. The hub serving those non-fixed users is used as the proxy location for scoring.

Hub

A site that provides educational or medical services to other sites but receives no end-user benefit. Pure hubs are usually listed but excluded from Fixed-project scoring.

End-User

A site where end users receive distance learning or telemedicine services, and where funded equipment is installed or placed. These sites count in Fixed-project scoring.

Hub/End-User

A site that both provides services/content and receives end-user benefit. These sites count in Fixed-project scoring.

Proxy Hub

A hub used as the scoring proxy for non-fixed end users in Hybrid or Non-fixed projects. Use only when the project includes mobile/home/changing end-user locations.

District Office

Usually a Hub if it manages or originates services. Change it to Hub/End-User only if the office also receives direct end-user benefit from the funded equipment.

Charter school eligibility guardrail

Practical answer: A public charter school can be a strong RUS DLT prospect or end-user site, but it is not automatically eligible solely because it is a charter school. Confirm the applicant structure, ownership of grant-funded equipment, SAM/UEI readiness, and whether the charter school is a public entity, nonprofit, part of an LEA, or part of a consortium.
QuestionWhat to check before treating it as eligible
Can the charter school apply directly?Only if it is a legally organized entity with authority to contract with RUS and own/manage the grant-funded assets.
Can it participate as an end-user?Yes, if the project delivers DLT-eligible educational or telemedicine services to rural residents/students and the site receives eligible equipment or direct benefit.
Could the district or consortium apply instead?Often yes. Use this when the charter is part of a broader K-12 or regional hub-and-spoke network.
What should reps avoid saying?Do not say “charter schools automatically qualify.” Say “charter schools may qualify depending on legal structure, project design, rurality, and documentation.”

Why a rural NCES Locale can still show D-1 review or 0

USDA DLT D-1 Rurality is not scored directly from NCES Locale. Locale 41, 42, or 43 is a helpful screening clue, but the DLT score depends on the 2020 Census town/place/CDP population and whether the site is in or contiguous to a non-rural area on the USDA DLT map.

Some school records use a postal city that is not the actual community where the campus sits. For example, a campus may have a Stephenville mailing address while the physical rural community is Huckabay. In that case, enter the correct value in Rurality Place/CDP, then re-run enrichment and verify the DLT map fields.

Update v20: K-12 district prospecting/search now uses the bundled ELSI/NCES K-12 School Districts 2025 CSV as the district source, including website and # Schools where available. School-site rows still use the NCES school layer by LEAID when adding all school buildings for a selected district.
